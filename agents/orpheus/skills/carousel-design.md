# Skill: Carousel Design for LinkedIn and Instagram

## Why Carousels Perform
LinkedIn and Instagram algorithms favour carousels because they generate "swipe" interactions that count as dwell time. A user swiping through 8 slides spends 4-5× longer with your content than they would with a single image post.

But only if the content earns each swipe.

## The Structure of a High-Performing Carousel

### Slide 1 — The Hook (the most important slide)
This slide must stop the scroll. It must make someone think "I want to see what is next."

Hook formulas:
- Bold claim: "Most [audience] are doing [X] wrong"
- Counterintuitive statement: "More features are making your product worse"
- Specific promise: "5 things every radiologist should know about AI accuracy"
- Provocative question: "Is your accounting software lying to you?"

Design requirements:
- Headline: maximum 8 words, large type
- No body copy on slide 1 — the headline IS the content
- Strong visual contrast
- Swipe indicator or "→" to signal there is more

### Slides 2-7 — The Content
One point per slide. No more.

Each slide structure:
- Slide headline (the point being made): 6-10 words
- Supporting detail: 2-3 short lines maximum
- Visual element: icon, diagram, data visualization, or image

Design requirements:
- Consistent layout across all slides (same grid, same font positions)
- Visual hierarchy: headline > supporting text > annotation
- Progress indicator: slide number or dot navigation
- Keep at least 20% white space — do not pack every slide

### Last slide — The CTA
What should the reader do now?

Options:
- Follow for more content like this
- Link in bio for the full resource
- Comment with their opinion
- Tag someone who needs to see this
- DM for [specific thing]

Never: "Like and share if you found this useful" — this is a relic of 2018

## Visual Design Rules

### Consistency
Every slide uses:
- The same background color (or consistent pattern)
- The same font pairing
- The same logo position (bottom right, 60% opacity)
- The same primary accent color
- The same margin/padding

If slide 3 looks completely different from slide 4, the carousel fails.

### Text on slides
- Headline: 36-48px, bold
- Body: 20-24px, regular
- Caption/annotation: 16-18px, light
- Never below 16px — test on a phone before publishing

### Platform specifications
- LinkedIn carousel: 1080 × 1080px (square), up to 10 slides (recommended: 6-8)
- Instagram carousel: 1080 × 1080px (square) or 1080 × 1350px (portrait), up to 10 slides

## Content Types That Work Well as Carousels

### The "X things" format
"7 signs your invoicing software is hurting your business"
Each slide = one sign. Works because it sets a clear expectation.

### The "how to" format
"How to reduce radiologist burnout with better AI integration"
Each slide = one step. Sequential logic that requires swiping to follow.

### The "breakdown" format
"Breaking down the €1M ARR milestone (the real numbers)"
Each slide = one part of the breakdown. Data-heavy but digestible.

### The "myth vs reality" format
Each slide = one myth (left side) and the reality (right side).
High engagement because it triggers agreement or disagreement.

## Figma Production Workflow

1. Create a frame at 1080×1080px (name it: "Carousel/[topic]")
2. Build slide 1 with all brand elements and layout
3. Duplicate for all remaining slides
4. Change only the content on each duplicate — structure stays identical
5. Use components for repeated elements (logo, slide number, background)
6. Export: File → Export → select all frames → PNG at 1× or 2×
